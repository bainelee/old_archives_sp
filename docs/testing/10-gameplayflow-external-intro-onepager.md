# GameplayFlow 对外介绍（1 页版）

## 一句话定位

GameplayFlow（GPF）是一套面向游戏开发阶段的测试、Debug、QA 工具：  
它可根据自然语言需求由 AI 生成可执行流程，在固定验证步骤下覆盖多种人为操作差异，并形成“发现问题 -> 定位归因 -> 修复复测”的闭环。

## 它解决什么问题

- 手工复测重复、耗时、易漏测。
- 改动后影响面不清晰，回归成本高。
- 不同成员复测同一问题，路径和结论不一致。

## 核心能力（当前已实现）

- **自动执行流程**：模拟常见玩家操作，覆盖核心玩法链路。
- **自然语言生成**：可从需求描述快速生成验证流程。
- **视觉联调验证**：支持截图对照、差异标识和复测闭环。
- **状态一致性校验**：可验证进度保存与恢复后的状态一致。
- **数据对账归因**：区分正常推进与采样时间差影响，减少误报。
- **过程可见**：步骤级播报“开始执行 -> 执行结果 -> 验证结论”。
- **结果可复盘**：输出报告、时间线、失败摘要、日志与截图证据。

## 为什么更可靠（通俗版）

- 全程可见：每一步都可追踪。
- 不易卡死：长时间无进展会自动停止并给出结论。
- 不易互扰：同一项目同一时间默认只跑一条主流程。
- 失败可解释：不是只给“失败”，还会给出可排查线索。
- 结果可复用：报告结构统一，便于团队沉淀经验。

## 典型案例（UI 同步联调）

在一次 UI 改版中，我们将设计同步到项目后，用 GPF 自动跑流程并产出关键截图，  
对照后标识差异区域，定位到具体步骤与模块，修正后再次复测并通过。  
结果是：在不增加手工回归成本的前提下，完成了可追溯的联调闭环。

## EXP 经验沉淀价值

GPF 内建 EXP（经验沉淀）机制：  
每次 Debug 收敛后沉淀“问题类型 + 调查路径 + 修复方案 + 回归标准”，  
让后续开发和同类问题排障都能复用，持续提升效率与稳定性。

## 适用边界

GPF 适合高频、标准化、可重复的开发期验证；  
不替代人工体验测试，也不覆盖完全未知路径的探索性验证。
