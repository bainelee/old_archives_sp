# 0 - 详细信息界面详解
## 0.1 details_panel结构和介绍

- details_panel

    - details_background: 详细信息的背景，保持和basebutton一样的ninepath切图，边距均为2px.会根据details_vbox_container的大小进行动态调整大小.

    - details_vbox_container,用于放置详细信息界面内容的垂直box,会根据自身内容的多少来调整大小.始终与四边距各距离2px.如此box最终尺寸为316x556,那么整个details_panel应该是320x560

        ### up_pointer 与 down_pointer

        - up_pointer和down_pointer:
            - 在details_vbox_container中垂直对齐,如果是up_pointer,则在容器的最顶端,如果是down_pointer,则在容器的最底端.

        ### details_title

        - details_title: 此信息的标题,以四个部分组成:

            - icon_detials_title : 尺寸为32*32,距离上方\左侧均为4px,icon资源使用相应icon,在details_title中的icon应为白色,通过调整原图hsv获得.与icon同一位置\在icon下层存在一个icon_frame_blue,因为没有代码逻辑所以不做单独解释.

            - text_title_name 名称 : 相应的名称,中文和英文稍有不同,组件内文字左对齐,与左侧的icon距离8px

                - 中文时,使用诸如 灾厄值\认知因子\意志因子\权限因子\计算因子\庇护能量\信息\研究员\住房\调查员\真相 的title.

                - 英文时,首先title为全大写,如为四种因子,则名称结构为FACTOR-TYPE,如认知因子:FACTOR-COGNITION.其他title为:庇护能量-SHELTER POWER,研究员-RESEACHER,信息-INFORMATION,住房-HOUSING,调查员-INVESTIGATOR,真相-TRUTH,灾厄值-CALAMITY VALUE

            - text_title_state 状态文本 :状态为当前当前资源或数值的状态,也可称之为等级,灾厄的等级和其他资源不同.英文状态同样使用大写,且文字右对齐,文字控件距离右侧8px.显示状态文本的数值\资源有:灾厄值,四种因子;不显示状态文本的资源\数值有:庇护能量,研究员,住房,信息,调查员,真相

                - 灾厄等级:
                    - 无 NULL : 0%
                    - 低危 MINIMAL : 1%-10%
                    - 劣势 WORSE : 11%-30%
                    - 险境 CHECK : 31%-80%
                    - 绝境 CHECKMATE : >= 81%

                - 资源状态:
                    - 枯竭 DRIED-UP : 此资源储备为0,且没有产出
                    - 稀缺 LACKING : 此资源储备为0,且预测中每周结余<0
                    - 紧张 STRAINED : 此资源储备>0,预测中每周结余<0(储备不断减少)
                    - 正常 NORMAL : 此资源储备>0但未满,预测中每周结余>0
                    - 富足 ABUNDANT: 此资源储备已满,预测中每周结余>0

            - detials_title_back 背景 :理论上大小为316x40的背景图,保持左右拉伸垂直对齐

        ### detail_storage_info

        - detail_storage_info: 资源的储存总览,拥有此条目的资源包括:四种因子\庇护能量\研究员\住房,此条目从上到下分为三部分.

            - text_storage_title 资源储存标题:
                - 四种因子的储存英文标题均为: Storage Server - Factor Name , 如Storage Server - Cognition Factor(此处和details_title逻辑不同).
                - 庇护能量storage title显示为: Shelter Power Output,中文为:庇护能量出力上限.
                - 研究员storage title显示为: Researcher Overview,中文为:研究员信息总览.
                - 住房storage title显示为: Housing Overview,中文为:住房信息总览.

            - storage_progress_bar 资源储存进度条:进度条总体组件理论尺寸为280x20,顶部距离 text_storage_title 8px.

                - 四种因子的进度条文字显示内容和进度:

                    - 当前因子储存量/因子服务器上限,如:45,000/50,000,因子服务器为因子储存的新专用词汇,在游戏中,因子由服务器储存,每种因子都自带一个服务器,在游戏中也可以新建额外的服务器来扩充因子储存量上限.此进度条与topbar上相应资源的progress_bar为同一套显示逻辑的不同尺寸表现.

                - 进度条的填充当前使用的是橙色纯色,在之后会替换为带有uv偏移动画的有方向标识的图像,在资源储备处于正数(即正在增加储备)时,图像使用增长对应图像,uv偏移方向为右侧,在资源储备处于负数(即正在消耗储备)时,图像使用降低对应图像,uv偏移方向为左侧.

                - 庇护能量出力上限的进度条特殊处理:

                    - 庇护能量出力上限的进度条上文字信息为:当前已分配庇护能量/庇护能量出力上限/庇护能量缺口(如有).

                    - 进度条信息为:底色\进度条总长度-庇护能量出力上限,橙色长度(从左向右)-已分配庇护能量/庇护能量出力上限,红色(从右向左)-庇护能量缺口数值/庇护能量出力上限.

                    - 庇护能量缺口的层级在已分配庇护能量之上,所以会存在被遮盖的情况.

                    - 如果庇护能量缺口数值超过出力上限(如缺口为1800,出力上限为1200),则红色进度条最多只占据90%进度,至少留出10%的空间来显示已分配能量.若已分配能量为0,则不保留这10%.

                    - 庇护能量缺口为另一个进度条(红色从右向左),与已分配庇护能量位置重叠
                        - 已分配庇护能量 : 已经分配给各区域的庇护能量数值
                        - 庇护能量出力上限 : 与当前的核心出力上限有关.
                        - 庇护能量缺口 : 数值为将所有 庇护状态处于 薄弱\暴露\绝境 的区域的庇护状态提升至 妥善的预计能量需求.

                - 研究员总览的进度条特殊处理:
                    - 研究员总览的进度条上文字信息为:闲置研究员数量/研究员总数/被侵蚀研究员数量.
                    - 研究员总览的进度条长度为:底色\进度条总长度-研究员总人数,橙色长度(从左向右)-闲置研究员数量/研究员总数,红色(从右向左)-被侵蚀研究员数量/研究员总数.

                - 住房信息总览的进度条特殊处理:
                    - 住房总览信息总览的进度条上文字信息为:住房需求总数/已提供的住房总数.
                        - 解析:由于住房是自动分配的,游戏中不存在玩家手动分配住房的情况,所以只要有能被满足的住房需求,就会自动被分配住房,因此,玩家需要关注的是'我还应该提供多少住房',所以在此处使用此种显示逻辑.
                    - 住房的进度条显示逻辑为:底色\进度条总长度-住房需求总数,橙色长度(从左向右)-可分配住房/住房需求总数(可能为0),红色长度(从右向左)-住房需求缺口(即住房需求总数-已提供住房数量)/住房需求总数.在存在住房需求缺口是,可分配住房应为0,因为不存在存在冗余住房却仍有住房缺口的情况.

                - 不显示detail_storage_info条目的详细信息:信息\调查员\真相:这三种资源不存在上限;灾厄值:灾厄值的进度条在bottom_bar上.

            - storage_warning 资源储存警告:由一个警告icon+警告文本组成.条目高12px,上方距离进度条条目8px
                - 四种因子-警告文本显示逻辑:
                    - 在资源状态为枯竭时,文字显示"资源已彻底枯竭,请尽快获得资源产出."文字为红色.
                    - 资源状态=稀缺 : "资源严重不足,请降低消耗或提高产出"
                    - 资源状态=紧张 : "产出不足,资源储备预计可供给{x}天." x=现有资源储备/此资源每日消耗储备数量.
                    - 资源状态=正常: "预计{x}天到达储存上限." x=资源储备空余/此资源每日填充储备数量.
                    - 资源储备=富足:"储存服务器已满"

                - 庇护能量-警告文本显示逻辑:
                    - 当存在庇护能量缺口时:"庇护能量缺口:{x}." x=庇护能量缺口数值
                    - 如无庇护能量缺口则不显示警告.

                - 研究员总览-警告文本显示逻辑:
                    - 研究员总览不显示警告文本

                - 住房信息总览-警告文本显示逻辑:
                    - 若存在住房缺口时,显示文本:"住房缺口:{x}."x=住房缺口数量.
                    - 无住房缺口时不显示警告.

        ### 资源消耗 list

        - 资源消耗list:存在资源消耗list的为三种因子:认知因子,意志因子,计算因子.显示逻辑稍有不同,权限因子没有资源消耗list,因为权限因子均为即使瞬时消耗.

            - 条目 total_expected_burn 预期总消耗:与上方距离8px,条目分为两部分,左侧为文字'预期总消耗'左对齐,右侧为预期总消耗的数值,靠右,右侧对齐.

                - 总消耗单位为每天,需要计算的消耗为两部分,分为「固有消耗」和「档案馆消耗」,其中特殊为计算因子,计算因子的第二部分为「核心消耗」,因为计算因子除了固有消耗之外,只有档案馆核心会进行消耗.

            - split_line 标准分割线,高度2像素,距离上一条目4px

            - 固有消耗组- fixed_overhead : 固有消耗是第一优先级的消耗,比如总的资源储量为300,固有消耗总数有200,档案馆消耗为200,那么资源会优先完全分配给固有消耗,剩余的100再分配给档案馆消耗(这当然会造成档案馆所需的资源不足,会造成其他后续结果.)

                - 固有消耗组距离上方8px,固有消耗title左侧为固有消耗的文字,右侧为固有消耗总数,单位为每天,如 200/天

                - 固有消耗基本为在探索中的探索节点带来的消耗(未来可能会有其他情况).每一个造成了消耗的节点会在详细信息界面添加一个固有消耗条目.

                - 固有消耗条目:每一个固有消耗条目高度为24像素,两个条目之间没有间距,每个条目包括条目文字和此条目每日消耗.条目文字的标准为 条目归属类型+条目名称,比如"探索-蓝木跃迁基地",如果条目归属来自房间,则条目归属类型显示房间类型,如图书室,最终形成的结构为"图书室-基础图书室"

            - split_line 标准分割线,高度2像素,距离上一条目4px

            - 档案馆消耗组 - archives_overhead : 档案馆消耗来自档案馆已建设区域的房间的消耗.
                - **标题层**: 显示"档案馆消耗" + 总计数值/天
                - **下属条目层**:
                    - 认知因子的消耗形式与其他有所不同,认知因子的档案馆消耗只有研究员消耗.条目仅有"所有研究员"一条,显示"所有研究员" + 消耗数值/天.即使消耗为0,此条目仍然显示.
                    - 同时下方会带有'消耗影响条目',如果研究员认知消耗发生变动,如提升或降低,会在此显示,高度16,带有一个warning_icon,文字形式为"增加原因/降低原因"+增加原因title,增加原因通常由决议\研究带来,示例为:增加原因:决议-「深度研究」.右侧为变化数值,通常为百分比(也可能是固定数值加减),如+20%.无影响时显示"无影响"占位文本.
                - 意志因子消耗条目,没有特别之处,和固有消耗条目显示方式一致.
                - 计算因子的消耗组没有条目,只有title层,显示如 核心消耗 400/天
                - 意志因子消耗条目,没有特别之处,和固有消耗条目显示方式一致.
                - 计算因子的消耗组没有条目,只有title层,显示如 核心消耗 400/天

            - split_line 标准分割线,高度2像素,距离上一条目4px

            - 产出组 output :产出可能来自于档案馆内的房间建设,也可能是来自于探索节点,title层级为所有产出的和.
                - 产出条目:结构为 产出来源类型-产出来源名称,如 图书室-基础图书室,探索-认知流节点

            - split_line 标准分割线,高度2像素,距离上一条目4px
            - 资源缺少/资源富余 resource_shortage/resource_surplus:
                - 在此资源的产出-消耗后小于零,则资源缺少,若大于零,则资源富余,资源缺少时会从资源储存中消耗资源(如有),资源富余时会将富余资源填充至储存(如未满).
            - bottom_placeholder 底部占位:最后一个条目(resource_shortage/resource_surplus 及其下的 resource_storage_row)之后,添加一个空的占位条目,高度20像素,用于底部留白或预留扩展.

        ### 庇护能量相关详细信息

        - 庇护能量相关详细信息:从上自下分别是 已分配庇护能量:显示已分配的庇护能量总数;固有分配组,显示被固有分配的庇护能量,下属有细则条目;建设分配:不显示细分条目;产出:由核心带来的庇护能量产出,可以没有细则(当前ui中额外写了一个档案馆核心产出条目);区域庇护状态组:显示当前档案馆里所有房间所处的庇护状态的数量.庇护状态从上到下分别是完美\妥善\薄弱\暴露\绝境\关停,后方的数字时处于这个庇护状态的已建设房间的数量.

        ### 研究员相关详细信息

        - 研究员相关详细信息:从上自下分别是:闲置研究员数量;被侵蚀研究员数量;在职研究员数量和分别在哪些区域的细则;研究员总数.

        ### 住房相关详细信息

        - 住房相关详细信息:从上自下分别是:可分配住房数量;已分配住房数量;住房缺口数量;住房产出细则;现有住房总数;

        ### 信息相关详细信息

        - 信息相关详细信息:从上自下分别是:
            - 信息产出（章节标题+总产出数值同行显示）
            - 细则条目（带灰色背景，显示产出来源+数量）
            - 额外影响（章节标题+总影响数值同行显示）
            - 细则条目（带灰色背景，显示影响来源+数量）
            - 信息储量（章节标题+储量数值同行显示）

        ### 调查员相关详细信息

        - 调查员相关详细信息:从上自下分别是:
            - 可分配调查员数量（章节标题+数值同行显示）
            - 已分配调查员数量（章节标题+数值同行显示）
            - 细则条目（带灰色背景，显示被分配到的探索节点名称+数量）
            - 已招募调查员数量（章节标题+数值同行显示）
            - 细则条目（带灰色背景，显示事务所区招募和事件/探索节点招募的来源+数量）

        ### 真相相关详细信息

        - 真相相关详细信息:从上自下分别是:
            - 已获得真相数量（章节标题+数量同行显示）
            - 细则条目（带灰色背景，显示每一个获得的真相名称）
            - 已解读真相数量（章节标题+数量同行显示）
            - 细则条目（带灰色背景，显示每一个已解读的真相名称）
