# EXP Overlay Input Debug

> 说明：本技能用于“覆盖层输入穿透”问题本身，不等价于 GF-EXP 分支编排流程。若用户说“执行GF-EXP流程”，应改用 `gf-exp-workflow`。

## 目标

将覆盖层/模态输入问题收敛为可重复流程：先复现，再定位，再最小修复，再回归固化。

## 快速流程（默认执行）

1. 固定三条约束（必须先写出来）
   - 覆盖边界：哪些区域应被覆盖，哪些区域必须保留交互。
   - 输入语义：哪些输入要被阻断，哪些输入必须透传。
   - 关闭路径：至少两条退出路径（主路径 + 冗余路径）。
2. 选择一个稳定状态量作为主断言
   - 优先业务状态/索引/开关位，避免高波动 UI 视觉状态。
3. 构建最小测试闭环
   - 打开覆盖层 -> 在关键区域点击 -> 关闭覆盖层。
4. 单变量迭代
   - 每轮只改一类变量：布局边界 / 输入门禁 / 测试断言 三选一。
5. 稳定性门槛
   - 连续至少 3 次通过后，再扩展次级断言。

## 盘查顺序（从快到慢）

1. 输入链入口
   - 是否在 `_input` 或统一 helper 中处理主交互。
   - 是否有早返回且未消费事件导致的二次消费风险。
2. 覆盖层几何与命中一致性
   - 视觉覆盖区域与实际命中矩形是否完全一致。
   - `mouse_filter` 与输入门禁是否互相打架。
3. 白名单与允许区
   - 是否存在多套“可点 UI 区域”规则，是否漂移。
4. 关闭机制
   - 关闭仅依赖单按钮时，是否会被覆盖层误伤。

## 修复策略（默认优先级）

1. 先做“边界清晰化”
   - 明确覆盖矩形，不用模糊“近似全屏”策略。
2. 再做“输入门禁最小化”
   - 仅对覆盖区阻断场景输入，不扩大到无关主 UI。
3. 最后做“测试去耦”
   - 主断言只保留低耦合指标；视觉断言作为补充。

## 反模式（禁止）

- 用“全局禁用主 UI”冒充防穿透。
- 视觉上有遮罩，但输入仍走后景逻辑。
- 一次同时改布局、输入、断言，导致责任面不可判定。

## 交付格式

每次处理此类问题，输出四段：

1. 现象与三条硬约束
2. 根因候选与证据
3. 最小改动与影响面
4. 回归结果（含连续通过次数）
