# GF-EXP 类型经验库

本文件只记录“问题类型级”方法，不记录具体 bug 事件。

## 类型 A：数值显示与运行态不一致

### 典型症状
- UI 显示为 0 或异常值，但逻辑预期为非 0。
- 同一资源在不同面板显示不一致。

### 最快定位路径
1. 先确认数据源配置（基础值/上限/缺省值）。
2. 再查运行态写入链（加载存档 -> set_resources -> UI 状态）。
3. 最后查 UI 渲染链（刷新时机、格式化、截断/Clamp）。

### 稳定断言
- 优先断言运行态状态键（资源字典、核心数值字段）。
- UI 文本断言只做补充，不做首轮主断言。

### 最小修复策略
- 优先修数据源与约束配置（避免在 UI 层打补丁）。
- 每轮只改一个责任面（配置 / 映射 / 渲染）。

### 反模式
- 用 UI 强制覆盖值掩盖底层配置错误。
- 一轮同时改配置、逻辑、测试，导致责任面不可复现。

## 类型 B：覆盖层输入穿透

### 典型症状
- 覆盖层在前景可见，但点击命中后景逻辑。

### 最快定位路径
1. 查输入入口（_input / helper 分发）是否有覆盖区门禁。
2. 查覆盖矩形与可视区域是否一致。
3. 查白名单规则是否在多模块漂移。

### 稳定断言
- 覆盖层状态 + 关键业务状态不变化（如选中索引）。

### 最小修复策略
- 先边界清晰化，再输入门禁最小化，最后补测试。
